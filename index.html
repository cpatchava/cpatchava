Cisco Systems

In my time at Cisco Systems I have worked on five different ASICs. My work includes both the Nexus 3500 series and the Nexus 9000 series. In my time here I learned from a very high level down to the very low transister level the full and very arduous process of making a hardware unit(from the transistors that shape it, to the software that utilizes it). I try to order the projects based on how close to the hardware level it comes to the high software level. A lot of these things I worked on overlap in time that I worked on them.


-Block Verification-
a. Owned verification on the Access Control block. I designed both a test plan and wrote code in C++, along with system Verilog to interweave the AC block to the test bench.
b. Wrote automation scripts to take pieces of code that could be reused (E.G top, driver, scoreboard, packet generator) and write them in a central location and regenerate most of my test bench even for each new chip
c. Templateized most of the code in C++ to handle decoders that would change from chip to chip so similar functionality APIs could be reused.

-Model/Design-
a. Wrote code in C++ to demonstrate what the RTL was supposed to be doing
b. Worked on the LED module in the nexus 3500 ASIC. System Verilog code to write the frequency with which the LED would toggle – based on the network traffic speed
c. Added flip flops on the block level to propagate a delay when pipeline valued were arriving too early.

-ASIC Drivers-
a. Once each chip came back, I would write APIs in C++ to configure and communicate between the ASIC and the oboard linux based OS.
b. Work with the team working on the Operating System for the ASIC to teach them how the block level APIs work, and write requested embedded code for talking to ASIC
c. Would also test the board when it came back. I would set tables and policers in the chip and pass traffic through to see if it actually behaves as expected.

-Regression-
a. responsible for regression on all blocks to verify that blocks were working as people reported.
b. Realized emails were the only way to track old regression results and progress so wrote a LAMP based backend stack to gather results of regression and automatically post latest results
c. Graphed data over time using Morris JS to show progress on a block by block level.
d. Wrote script to catch build errors and send automatic notifications to users relevant to the problem – saved the time of having to wait until all the results were tabulated.

-Synthesis-
a. Worked with vendor to take RTL code and get it compiled into a gate level synthesis netlist used to run spyglass manually to check to see if the files we have match the synthesized netlist sent back from vendor
b. Automating the backend to see results of synthesized netlists, rather than having to run it manually every time a new netlist came in (usually weekly).
c. Scripted a means to check the output results of the synthesis to see if new netlists were clean – parsed output files to see if results compiled clean.
d. Used pattern recognition to remove ignorable warnings/push high level problems and parsers would help diagnose potential issues from previous similar errors

Red v Blue

For the last year we have been seeing an influx of campaign promises and articles from various news organizations about the merits and demerits of one candidate or the other. I have noticed, especially with the whole Bernie Sanders movement and their frustration in how the media has been disproportionately bringing light to certain candidates over others that there is quite a media bias towards one candidate or the other. This is something that first came to light to me from watching The Daily Show with Jon Stewert. I am very unhappy with the fact that most news media has a bias one way or the other, a lot of times their alligences leaning towards their party affiliations.

I wanted to make this system a little easier for people who are genuienly interested in politics and also do not want to be swayed by the news organizations they represent. In order to rectify this I started to work on an idea called Red V Blue. It's a project meant to demonstrate the bias one way or the other (democratic of republican) as you are reading the article.
